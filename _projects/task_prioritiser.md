---
title: "Task Prioritiser"
github_link: "https://github.com/AE-CuriousSolver/TaskPrioritiser"
what: "A backend-focused task management application built around CRUD operations and
  priority-based task ordering. Tasks include optional deadlines and can be marked as completed.
  Each task is assigned effort, impact, and urgency values, which are used to compute a priority score.
  The system supports filtering for outstanding tasks, including a 'due today' view."
purpose: "This project was built as a practical exercise to re-familiarise myself with backend development, particularly API design, domain modelling, and service layer architecture.
  The emphasis is intentionally on learning and iteration rather than optimisation or production readiness."
learnings: 
 - "Prioritising core functionality first helped avoid getting lost in over-engineering or 'interesting but unnecessary' features."
 - "Using IDE AI assistance (e.g. IntelliJ AI chat) was useful not just for solutions, but for understanding trade-offs and reasoning behind approaches."
 - "Local development setup was a significant friction point when working across environments and unfamiliar tooling, highlighting the importance of reproducible dev environments."
challenges:
  - "Initial setup and environment configuration took longer than expected and impacted early momentum."
  - "Balancing structure vs simplicity when designing the domain model."
  - "Avoiding feature creep while still allowing space for exploration."
futures: 
  - "Add a lightweight frontend to interact with the API and improve usability."
  - "Introduce database persistence to move beyond in-memory or basic storage."
  - "Explore deployment to a local or cloud environment for end-to-end workflow practice."
  - "Refine prioritisation logic and test how useful it is in real task scenarios."
